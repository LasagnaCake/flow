FLOW: Fast Lazy Object Writing

"Lazy objects for lazy people"

1. What is flow?

JSON, but simple(r).

Take the following JSON example:

  "orders": [     {       "order_id": 0,       "price": 26.99,       "customer_id": 0,       "items": [       ],     }   ],

Here's the same, in FLOW:

  "orders": [     {       "order_id": 0,       "price": 26.99,       "customer_id": 0,       "items": [       ],     }   ],

But wait – it can be made even simpler! Look now:

  orders [     {       order_id 0       price 26.99       customer_id 0       items [       ]     }   ]

Much better, isn't it?

Simplicity Meets Functionality

The goal of FLOW is simple – Make JSON easier to write, while also maintaining backwards compatibility with it, and adding along some niceties.

As such, FLOW has a few differences from its parent:

  • Delimiters (: and ,) are optional
  • "Identifiers" (Strings composed solely of alphanumeric characters (NOT beginning with a number) and underscore (_)) can be unquoted
  • Native binary data support (via Encoded Entities)
  • Line and block comment support

Additions over JSON

Encoded Entities

Encoded Entities are the way to store binary data in a FLOW file, in an editable way.

Essentially, it is a base-N encoded string, with N being a power of 2.

Specified as follows:

! <base> "<content>"

Where <base> is the base of the string, and <content> is the encoded content. Spaces and separators are optional, but not recommended.

Example of the ASCII text "Hellow, FLOW!" stored in a base-64 string:

!64"SGVsbG8sIEZMT1ch"

While less space-efficient than a proper binary encoding, this allows it to be URL-safe and human-editable.
